Pro_Pack : Exporter¶

ProPack exporter module is our own export management system for handling the output of files based on the metaData export node class systems. This module is largely the UI management as the majority of the codebase is bound to the metadata_pro mNode classes

>>> # import statement for the module via the r9pro decompiler
>>> from Red9.pro_pack import r9pro
>>> r9pro.r9import('r9exp')
>>> import r9exp

export_fbx(runSceneRoot=False, mode='simple', respect_settings=True, *args, **kws)¶

run the fbx anim transfer export process

Parameters:
  • runSceneRoot – run the sceneRoot process during the export, default = False
  • mode – ‘simple’ (default) or ‘selection’ 2 different ways to produce the fbx data. ‘simple’ creates a duplicate bound skeleton and exports that data, giving you just the base skeleton output, aimed at animation data ‘selection’ exports the rigs nodes directly, including the skins and blends, whilst the data included will be minimum, skeleton, mesh etc, it will always include the rigs upper hierarchy in the data
  • respect_settings – do we respect and use the fbx_settings from the base UI or ignore them?
